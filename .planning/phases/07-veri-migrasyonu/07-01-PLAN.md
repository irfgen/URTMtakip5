<task type="auto">
  <name>Task 1: Create migration script</name>
  <files>backend/scripts/migrate-to-postgresql.js</files>
  <action>
Create migration script that:
1. Connects to SQLite (read-only) using better-sqlite3
2. Discovers all tables via `SELECT name FROM sqlite_master WHERE type='table'`
3. For each table:
   - Reads schema via `PRAGMA table_info`
   - Creates matching PostgreSQL table using Sequelize sync or raw SQL
   - Copies data in batches of 1000 rows using transactions
4. Handles type mapping:
   - INTEGER PRIMARY KEY → BIGSERIAL
   - INTEGER → INTEGER or BIGINT (based on size)
   - TEXT → TEXT
   - REAL → DOUBLE PRECISION
   - BLOB → BYTEA
5. Preserves foreign key relationships (disables FK checks during migration)
6. Uses `DB_DIALECT=postgresql` for target connection via existing database.js
7. Logs progress with row counts
8. Commits each batch, rolls back on failure

Use better-sqlite3 for SQLite read (npm install if needed), sequelize for PostgreSQL write.
  </action>
  <verify>
node backend/scripts/migrate-to-postgresql.js --dry-run 2>&1 | head -50</verify>
  <done>Migration script created, prints table list without executing</done>
</task>

<task type="auto">
  <name>Task 2: Create verification script</name>
  <files>backend/scripts/verify-migration.js</files>
  <action>
Create verification script that:
1. Connects to both SQLite and PostgreSQL
2. For each table: compares row counts
3. For each table: calculates checksum of primary key + string columns (MD5)
4. Reports discrepancies with table name, expected vs actual counts
5. Exits with code 1 if any mismatch

Checksums calculated via: SELECT COUNT(*), GROUP BY primary_key ORDER BY primary_key
</action>

</task>
